Batch Reconciliation Queue

Version: 18.0.2.0.0

Author: Quanam

Overview

High-performance solution for bank reconciliation of batch payments with more than 5,000 records. Designed to work within Odoo.sh's 15-minute timeout limit by dividing work into independent chunks processed via queue_job.

Key Benefits:
  • Process 5,000+ payments without timeouts
  • Each chunk completes in <10 minutes (Odoo.sh compatible)
  • Individual retry on chunk failure
  • Real-time progress tracking
  • Full accounting integrity maintained

Architecture

Component Description
Master Job Coordinates the entire reconciliation process, divides work into chunks, tracks overall progress
Chunk Job Processes ~500 payments independently, can be retried individually on failure
Queue Channel Dedicated root.reconciliation channel controls concurrency

Processing Flow

  1. Detection: User validates reconciliation with >500 payments
  2. Master Creation: System creates a master job and divides work into chunks
  3. Chunk Processing: Each chunk is queued and processes ~500 payments
  4. Finalization: After all chunks complete, final reconciliation runs
  5. Notification: User is notified of completion or any failures

Performance

Payment Count Chunks Expected Time Status
<500 Standard flow <30 seconds Synchronous
500-2,000 1-4 2-8 minutes Background
5,000 10 10-15 minutes Background
9,000 18 18-25 minutes Background

Configuration

Configuration parameters (Settings → Technical → Parameters):

Parameter Default Description
batch_reconciliation_queue.queue_threshold 500 Minimum payments to trigger background processing
batch_reconciliation_queue.chunk_size 500 Payments per chunk (keep <700 for Odoo.sh)
batch_reconciliation_queue.max_parallel_chunks 3 Maximum concurrent chunks (higher = faster but more resources)

SQL Optimizations

The module creates the following indexes during installation:

  • account_move_line_move_display_type_idx - Payment term line lookups
  • account_payment_batch_state_idx - Batch payment filtering
  • account_move_line_account_reconciled_idx - Reconciliation searches
  • account_bank_statement_line_move_idx - Statement line lookups

Monitoring

Access the job monitor via: Accounting → Customers → Reconciliation Jobs

Each job shows:

  • Overall progress percentage
  • Chunk-by-chunk status
  • Processing duration
  • Error details (if any)
  • Related queue jobs
Important: This module requires queue_job with a running job runner. On Odoo.sh, the job runner is managed automatically. For on-premise installations, ensure the connector channel worker is running.
